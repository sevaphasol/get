# Инструкция по работе с ветками в Git

## 1. Получение ветки с сервера

```bash
# Сначала убедись, что у тебя есть все последние изменения с удаленного репозитория
git fetch --all

# Посмотри список всех веток (локальных и удаленных)
git branch --all

# Если ветка новая и ее нет локально:
git checkout -b имя_ветки origin/имя_ветки

# Если ветка уже есть локально, просто переключись на нее:
git checkout имя_ветки

# Обнови локальную ветку (если нужно)
git pull origin имя_ветки

# Перейди в ветку, В КОТОРУЮ хочешь влить изменения (например, main)
git checkout main

# Влей изменения из другой ветки (например, feature-branch)
git merge feature-branch

# Если возникли конфликты:
# 1. Открой файлы с конфликтами и исправь их
# 2. Добавь исправленные файлы:
git add имя_файла
# 3. Продолжи слияние:
git merge --continue

# 1. Убедись, что все работает (запусти тесты если они есть)
# 2. Запуши изменения на сервер:
git push origin main

# 3. Если ветка больше не нужна, можно удалить:
# Локально:
git branch -d feature-branch
# На сервере:
git push origin --delete feature-branch
```
